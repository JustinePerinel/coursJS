JS - II 03 Les types de données JavaScript

1 - Sept types de données JavaScript

  1. String ou "chaîne de caractères"
  2. Number ou nombres
  3. Boolean ou "Booléen"
  4. Null ou "vide"
  5. Undefined ou indéfini
  6. Symbol ou "symbole"
  7. Object ou "objet"

Pour voir les types de données on utilise la fonction typeof()

1 - String

Sur les guillemets et apostrophes droits et sur l'échappement


En JavaScript, on va pouvoir utiliser indifféremment des guillemets droits ou des apostrophes pour entourer une chaine de caractères et ces deux méthodes vont être strictement équivalentes à la différence d'autres langages comme le PHP par exemple.

Faites attention cependant à un point : si votre chaine contient un caractère qui est le même que le délimiteur de chaine choisi, il faudra neutraliser ce caractère en l'échappant au moyen d'un antislash ou changer de délimiteur.

2 - Number

Les variables en JavaScript vont également pouvoir stocker des nombres. Des nombres entiers positifs ou négatifs ou à virgule (décimaux)

3 - Booleen (boolean en anglais)

Ce type de données sera utile pour vérifier si ce test est vrai ou faux

4 et 5 - Null

Null et Undefined sont des types un peu particuliers

  • null : absence de valeur connue.
  • undefined : variable non définie.

5 - Undefined

Si on déclare une variable sans lui attribuer de valeur, alors son type sera Undefined

6 - Symbol et 7- Object

Les objets sont des structures complexes qui vont pouvoir stocker plusieurs valeurs en même temps et que nous étudierons plus tard car il s'agit là d'un sujet complexe à appréhender pour un débutant

Si on déclare une variable et qu'on lui passe null, alors son type sera Object
