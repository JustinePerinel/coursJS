II - 01 Les variables

1 - Qu'est ce qu'une variable ?

Une variable est un conteneur servant à stocker des informations de manière temporaire, du texte (une chaîne de caractères) ou un nombre. La variable varie ! Elle stocke différentes valeurs au fil du temps.

Une variable est la valeur sont deux choses différentes

2 - Règles de déclaration

On dit (comme en php) que l'on "déclare" une variable. On déclarer une variable avec des mots-clefs.

A - Les mots-clefs

  • let
  • var


B - Quelques règles

  • Le nom d'une variable doit obligatoirement commencer par une lettre ou un underscore _
  • Le nom d'une variable ne doit contenir que des lettres, des chiffres et des underscores, pas de caractères spéciaux
  • Il n'y a pas d'espaces
  • cf. introduction.js

3 - Initialiser une variable

Quand on assigne une valeur à une variable vide, on dit que l'on stocke une valeur ou qu'on initialise une variable

4 - Différences entre "var" et "let"

let est plus récent que var. On utilisera par défaut let. Var est l'abréviation de variable. On peut écrire le code en une ligne ou en deux (cf. introduction.js). Attention à l'ordre des données !

  • A. La remontée ou hoisting des variables

    Avec var, on n'est pas obligé de déclarer la variable avant de la manipuler ou l'affecter. cf. introduction.js : ligne 26.

  • B. La redéclaration des variables

    Avec var on avait le droit de déclarer une même variable en utilisant à chaque fois var, ce qui avait pour effet de modifier la valeur stockée. On ne peut pas le faaire avec let. cf. introduction.js : ligne 45.

  • C. La portée des variables

    La portée d'une "variable" désigne l'endroit où cette variable va pouvoir être utilisée dans un script. Nous reparlerons donc de cette portée des variables lorsque nous aborderons les fonctions en JavaScript.

    let est plus contraignant que var. Le premier à une portée locale, alors que le second à une portée globale.

  • CONCLUSION : let ou var ?

    La syntaxe de déclaration des variables avec let correspond à la nouvelle syntaxe. La syntaxe avec var est l'ancienne syntaxequi est vouée à disparaître. Vous devriez donc aujourd'hui toujours utiliser le mot clef let pour déclarer vos variables, sachant qu'il est parfois contraignant.

5 - Quelle utilité pour les variables ?

Il sera pratique de stocker différents types de données ou d'informations dans une variable.

Exemple : avec la fonction prompt() on récupérera une donnée envoyée par un utilisateur, on fera un script qui contiendra une variable qui stockera la ou les données quand elles seront envoyées. cf introduction.js : ligne 41.
