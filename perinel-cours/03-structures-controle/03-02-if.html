III. 02 - If...

1 - If... : utilisation

La condition if est l'une des plus utilisées et est également la plus simple à appréhender, puisqu'elle va nous permettre d'exécuter un bloc de code, si et seulement si le résultat correspond au booléen true.

Deux "if" exécutés dans if.js

  • Cette alerte va s'afficher car on a auparavant déclaré let x =4 ;
    On aura donc une fenêtre d'alerte qui s'affiche avec le string concerné.
    if(x > 1){
    alert('x contient une valeur supérieure à 1');
    }
  • Cette alerte ne va pas s'afficher car on a auparavant déclaré let x =4 ; et que let y = 0;
    Ce qui signifie que les valeurs ne sont pas égales.
    if(x == y){
    alert('x et y contiennent la même valeur');
    }

if(y){
alert('la valeur de y est évaluée à true');
}
Ici, l'alerte ne s'affiche pas car let y = 0;. Or, tout test d'une condition va être évalué dans un contexte booléen, donc à true. A l'exception des valeurs suivantes, qui vont être évaluées à false
  • Le booléen false
  • La valeur 0
  • Une chaîne de caractère vide ''
  • La valeur null
  • La valeur undefined
  • La valeur NaN (Not a Number)

2 - If... : Inverser la valeur logique d'un test

Dans un if, le code placé dans le if n'est exécuté que si le résultat de la comparaison est true.
Dans certaines situations, on préfère vérifier si la comparaison est false. On peut vérifier que c'est bien faux en comparant explicitement le résultat de la comparaison à false. Voir if.js, ligne 21.
