La place du code et l'ordre d'exécution

L'ordre d'exécution du code par la navigateur

Par défaut, le navigateur va lire et exécuter le code dans l'ordre de son écriture. Lorsque le navigateur arrive à un script, il va stopper le traitement du reste du HTML jusqu'à ce que le code Java Script soit chargé dans la page et exécuté.

Résumons ce qui se passe dans ce cas

  1. La navigateur commence à analyser ou à traiter le code HTML.
  2. Le navigateur rencontre un élement script.
  3. Le contenu JS est demandé et téléchargé (s'il est en externe) puis exécuté. Durant tout ce temps, l'analyseur du navigateur bloque l'affichage du HTML ce qui peut ralentir, si le script est long l'affichage de la page.
  4. Dès que le JS a été exécuté, le contenu HTML est affiché.

Ce soucis d'attente est résolu grâce au téléchargement asynchrone des données avec l'attricut async ou defer dans le cas où l'in fait appel à des fichiers externes.



Le point virgule

Le point virgule ; est utilisé pour signifier la fin d'une instruction (une variable, une instruction, une boucle, etc.). On le mettra à la fin de presque toutes les instructions (il y a des exceptions).
